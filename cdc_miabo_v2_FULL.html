Présentation & Proposition de Valeur
Contexte, vision stratégique et périmètre de la refonte

1.1 Contexte & Genèse du Projet

MIABO est une plateforme malgache de tutorat personnalisé, née du constat que 78% des élèves malgaches à besoins de soutien scolaire n'y ont pas accès faute de disponibilité, d'information ou de moyens financiers adaptés. La v1 (React + Vite + Supabase) a validé les concepts clés : matching tuteur-élève, tests VARK, messagerie. La v2 constitue une refonte totale vers une architecture production-ready (Next.js 14 + FastAPI + Supabase), PWA mobile-first, bilingue français/malgache.

1.2 Problèmes Résolus

🔍
Accès opaque aux tuteurs

Aucun annuaire structuré. La v2 introduit une recherche géolocalisée avec matching intelligent (VARK + style pédagogique + disponibilités).

💳
Paiement informel non sécurisé

Les règlements se font en cash. La v2 intègre MVola & Orange Money avec escrow et facturation automatique.

👨‍👩‍👧
Parents exclus du suivi

Pas de visibilité sur la progression. La v2 donne aux parents un dashboard dédié avec validation et paiement.

📚
Ressources pédagogiques dispersées

CANOPE Madagascar produit des ressources non centralisées. La v2 intègre un espace ressources certifié.

🧭
Orientation scolaire absente

Le COSP n'a pas de canal numérique. La v2 intègre bilans d'orientation et recommandations tuteurs.

📶
Faible bande passante

Réseau 2G/3G majoritaire. La v2 est PWA avec cache offline, images compressées et lazy loading agressif.

⚠️ Décision d'architecture critique
La migration de Vite vers Next.js 14 nécessite une réécriture complète du routage (React Router → App Router). Le contexte d'auth (UserContext.tsx) sera remplacé par Supabase Auth + middleware Next.js. Les composants UI peuvent être réutilisés avec adaptation des imports.

3.3 Sécurité & Row Level Security (RLS)

Principe de sécurité fondamental
Toutes les tables Supabase ont RLS activé. Aucune table n'est accessible sans politique explicite. Les données des mineurs bénéficient d'une protection renforcée. Le backend FastAPI valide le JWT Supabase sur chaque requête.
TablePolitique RLSRègle
profilesSELECT, UPDATEUtilisateur voit/modifie uniquement son propre profil. Admin peut tout voir.
tutor_profilesSELECT public, INSERT/UPDATE propriétaireProfil validé = visible par tous. Non validé = tuteur + Admin/CANOPE seulement.
sessionsSELECT élève+tuteur+parent, INSERT élève/parentSession visible uniquement par les participants (élève, tuteur, parent lié).
messagesSELECT participants, INSERT participantMessage visible uniquement par les participants de la conversation.
paymentsSELECT payeur+tuteur+AdminTransaction visible par le payeur, tuteur bénéficiaire et Admin.
minor_profilesSELECT parent+tuteur actif+AdminDonnées élève mineur visibles uniquement par le parent lié ET tuteur avec session active.
notificationsSELECT destinataire uniquementNotification strictement privée.
admin_*SELECT/UPDATE Admin seulementTables d'administration inaccessibles sans rôle Admin.

Exemple politique RLS — table sessions

-- SELECT : visible par les participants directs
CREATE POLICY "sessions_select_participants"
ON sessions FOR SELECT
USING (
  auth.uid() = student_id
  OR auth.uid() = tutor_id
  OR auth.uid() IN (
    SELECT parent_id FROM parent_student_links
    WHERE student_id = sessions.student_id
  )
  OR EXISTS (
    SELECT 1 FROM profiles
    WHERE id = auth.uid() AND role = 'admin'
  )
);

-- INSERT : eleve ou parent uniquement
CREATE POLICY "sessions_insert_student_parent"
ON sessions FOR INSERT
WITH CHECK (
  auth.uid() = student_id
  OR auth.uid() IN (
    SELECT parent_id FROM parent_student_links
    WHERE student_id = sessions.student_id
  )
);

Glossaire
Termes métier, techniques et réglementaires utilisés dans ce CDC

8.1 Termes Métier

MIABO
Plateforme malgache de tutorat personnalisé. Le nom évoque le soutien, l'accompagnement et la croissance en malgache.
Tuteur
Enseignant ou étudiant avancé proposant des sessions de soutien scolaire individuel ou en petit groupe. Doit être validé par CANOPE/Admin avant de pratiquer sur la plateforme.
Élève
Utilisateur inscrit sur MIABO pour recevoir un soutien scolaire. Peut être mineur (nécessite validation parentale) ou majeur.
CANOPE [MODIFIÉ]
Réseau de Création et d'Accompagnement Pédagogique. Rôle intégré à MIABO donnant accès à un dashboard dédié pour administrer les tests VAK/RIASEC/DISC, gérer les jeunes (Option A : élèves MIABO, Option B : jeunes externes), envoyer du contenu de formation et planifier des séances. Identifié par un code SESAME format CANOPE-XXXX.
COSP [MODIFIÉ]
Centre d'Orientation Scolaire et Professionnelle. Rôle intégré à MIABO héritant de toutes les fonctionnalités CANOPE, avec en plus la capacité de rédiger des bilans d'orientation personnalisés, recommander des tuteurs et partager des rapports avec les élèves et parents. Code SESAME format COSP-XXXX. Profil inclut les dates de formations COSP.
SESAME [AJOUTÉ]
Système d'identification des acteurs CANOPE et COSP. Attribue des codes uniques (CANOPE-XXXX, COSP-XXXX) requis à l'inscription sur MIABO pour activer les droits correspondants. Fournit également la charte graphique appliquée sur les rapports de tests.
VAK [AJOUTÉ]
Visuel, Auditif, Kinesthésique. Test de style d'apprentissage basé sur 30 affirmations (Oui/Non). Administré par CANOPE/COSP. Remplace l'ancien test VARK (sans la dimension Lecture/Écriture). Source : document SESAME "Test VAKOG - Style d'apprentissage".
DISC [AJOUTÉ]
Profil de communication en 4 dimensions : Dominant (D), Influent (I), Stable (S), Consciencieux (C). Test administré par CANOPE/COSP en 3ème position après VAK et RIASEC. Questions reproduites visuellement depuis le document scanné SESAME.
Option A / Option B [AJOUTÉ]
Deux modes de création de test pour CANOPE/COSP. Option A : le jeune est un élève déjà inscrit sur MIABO (profil pré-rempli, résultats visibles par l'élève et son parent). Option B : le jeune n'a pas de compte MIABO (saisie manuelle complète, résultats visibles uniquement par CANOPE/COSP).
Session
Séance de tutorat planifiée entre un élève et un tuteur. Peut être en présentiel ou en visioconférence. Durée : 1h à 3h.
Escrow
Mécanisme de séquestre des fonds. Le paiement est retenu par MIABO entre le moment du paiement et la confirmation de fin de session, protégeant les deux parties.
MVola
Service de mobile banking de Telma Madagascar. Principal moyen de paiement mobile au Madagascar.
Orange Money
Service de mobile banking d'Orange Madagascar. Deuxième opérateur de mobile money après MVola.
Ariary (Ar)
Monnaie nationale malgache. Toutes les transactions MIABO sont en Ariary. 1 EUR ≈ 4 700 Ar (2026).
Score de compatibilité
Pourcentage (0-100%) calculé par l'algorithme MIABO indiquant l'adéquation entre le profil d'apprentissage d'un élève et la méthode pédagogique d'un tuteur.
KYC
Know Your Customer. Vérification d'identité des tuteurs avant qu'ils puissent effectuer des virements de leur wallet MIABO.

8.2 Termes Techniques

VARK
Visual, Auditory, Read/Write, Kinesthetic. Modèle de styles d'apprentissage de Neil Fleming utilisé pour le matching élève-tuteur sur MIABO.
RIASEC [MODIFIÉ]
Modèle d'orientation de John Holland (Réaliste, Investigateur, Artistique, Social, Entreprenant, Conventionnel). Sur MIABO, le test RIASEC comprend 66 activités (11 par lettre). Administré par CANOPE/COSP en 2ème position. Le résultat est un code à 2 lettres (ex: "SE"). Source : "Test RIASEC U-MAGIS" SESAME.
RLS
Row Level Security. Mécanisme PostgreSQL/Supabase permettant de définir des règles d'accès ligne par ligne. Garantit qu'un utilisateur ne voit que les données autorisées.
SSR / SSG / ISR
Server-Side Rendering / Static Site Generation / Incremental Static Regeneration. Modes de rendu Next.js 14 choisis selon le besoin SEO et la fraîcheur des données.
PWA
Progressive Web App. Application web installable sur mobile comme une app native, fonctionnant partiellement hors connexion grâce au Service Worker.
JWT
JSON Web Token. Token signé utilisé par Supabase Auth pour authentifier les requêtes API. Validé par FastAPI à chaque endpoint protégé.
Supabase Realtime
Fonctionnalité Supabase permettant d'écouter les changements de BDD en temps réel via WebSocket. Utilisé pour la messagerie, notifications et disponibilités.
App Router
Système de routage de Next.js 14 basé sur le système de fichiers dans le dossier app/. Remplace le Pages Router des versions antérieures.
next-intl
Bibliothèque d'internationalisation pour Next.js App Router. Gère les traductions FR/MG, la détection de langue et le routage localisé.
APScheduler
Bibliothèque Python pour la planification de tâches. Utilisée dans FastAPI pour les rappels de sessions, déblocages d'escrow et rapports mensuels.
tsvector
Type PostgreSQL pour la recherche full-text. Utilisé sur les profils tuteurs (nom, matières, bio) pour la recherche instantanée.

8.3 Abréviations du CDC

CDC
Cahier des Charges. Document de référence définissant les besoins fonctionnels et techniques d'un projet informatique.
MVP
Minimum Viable Product. Version minimale de la plateforme suffisamment complète pour être lancée. Correspond aux 6 premiers sprints (3 mois).
MoSCoW
Must/Should/Could/Won't. Méthode de priorisation des fonctionnalités utilisée en gestion de projet agile.
RBAC
Role-Based Access Control. Chaque rôle MIABO (élève, tuteur, parent, admin, canope, cosp) a des permissions distinctes.
KPI
Key Performance Indicator. Sur MIABO : taux d'occupation tuteurs, taux de complétion sessions, NPS, chiffre d'affaires mensuel.
